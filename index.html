Use Case диаграмма процессов создания системы:



Use Case диаграмма процессов использования системы:


1. Идентификатор прецедента — ПО для 1C-Предприятие.
2. Название прецедента — Разработка программы по поддержке работы call-центра.
3. Контекст — Call-центр.
4. Участники (actors) и цели (goals).
Участник Категория Цель
Заказчик (call-центр) Внешний Предоставить требования к ПО
Аналитик (менеджер кол-центра) Основной Составить ТЗ по требованиям
Разработчик Основной Создать ПО по поддержке работы call-центра
1с конфигуратор для написания программы Инструмент Предоставляет возможность выполнения функции разработки, отладки, обновления и администрирования
Встроенный язык программирования 1 с предприятия Инструмент Предоставляет возможность разрабатывать приложения на базе платформы 1С-Предприятие

5. Предусловия
  • Заказчик сформировал начальную концепцию системы и подготовил требования к ПО;
  • Система 1С-Предприятие установлена на ПК;
  • Аналитик и разработчик ознакомлены с функционалом системы;
  • Освоено владение инструментарием 1С-Предприятия для разработки.

6. Постусловия
  • Разработана программа по поддержке call-центра;
  • Реализована возможность регистрации звонков и ведения журнала;
  • Требования заказчика удовлетворены.


7. Основной поток (main flow)
Участник Действие Ожидаемый результат
Аналитик Составляет ТЗ в котором отражены параметры работы ПО Готовое ТЗ
Разработчик Создание информационных структур в БД 1с Готовые информационные структуры в БД 1с
Разработчик Создание форм ввода-вывода 1с Готовые формы ввода-вывода 1с
Разработчик Создание запросов к БД для регистрации звонков и отображение журналов Готовые запросы
Разработчик Реализация ПО на основании ТЗ Готовое ПО

8. Исключения (exceptions).
  • Недостаточный уровень компетенции IT-специалистов;
  • Отмена заказа к разработке ПО.

9.Альтернативы (alternates)
  • Изменение требований заказчиком, по причине избыточности функционала и сложностью в освоении.
